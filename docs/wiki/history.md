# History

Every finished session is kept, whether it ended in a shipped PR or a downloaded report.

## Where it is

Side panel → **History** tab. Shows your last 10 sessions, newest first.

## What's on each entry

- The mode used (copy/debug/none) and a one-line summary of the intent.
- Which outcome it had — Shipped (with PR link) or Downloaded report.
- Timestamp and the page it ran on.

## Opening one

Click an entry → read-only replay: the conversation, the edits made, and (for debug sessions) the findings. You can't continue chatting inside a replay — it's a record, not a live session.

## Re-downloading

If a session's report was downloaded once and you lost the file (or want it again in a different tool), re-download it straight from its History entry — no need to re-run the session.

## PR links

Shipped sessions keep their PR link live in History — click through to check status or review, even after the panel's live task timeline has scrolled away.

## Deleting

Each entry has a delete action if you want to clear it out early. Otherwise the ring buffer keeps the last 10 automatically — the 11th push drops the oldest.

## What's not kept

- Nothing beyond 10 sessions — this is a quick-reference ring buffer, not a full audit log or a searchable archive.
- No page content is retained beyond what's already in the recorded changeset/report for that session — see [Privacy & keys](privacy-keys.md) for exactly what that includes.
